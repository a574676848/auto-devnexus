# Open Source Docs Reference

## 一、写“优秀开源项目文档”时真正要解决的问题

很多仓库并不是没有文档，而是文档的角色错位了。最常见的问题不是“内容太少”，而是：
- README 像开发周报
- docs 缺少入口页和清晰分层
- examples 已经变了，文档还停留在旧版本
- 图片、致谢、许可证说明存在，但没有被正确接入
- 英文术语很多，中文团队难以高效阅读

这个 Skill 的方法论，就是把这些错位逐个纠正。

## 二、推荐工作流

### 1. 盘点资产
先列出：
- 根 README
- `docs/` 目录
- `docs/images/` 或其他图片目录
- `examples/`
- `LICENSE`
- `CONTRIBUTING.md`
- `CODE_OF_CONDUCT.md`
- 致谢、上游项目说明、第三方协议说明

### 2. 判断每类内容该放哪里

| 内容类型 | 更适合的位置 |
|---|---|
| 项目定位、核心价值、快速开始 | 根 README |
| 系统架构、设计原则、概念说明 | `docs/` |
| API / CLI / 配置细节 | `docs/` 参考文档 |
| 最小可运行样例 | `examples/` |
| 贡献方式、行为准则、许可证 | 仓库根目录 + README 链接 |

### 3. 重写而不是堆叠
如果原文是“本次迭代新增了 X、修复了 Y”，要把它改写成稳定表达：

不推荐：
```markdown
本次迭代新增了四种接入方式，并补充了若干文档。
```

推荐：
```markdown
项目提供四种接入方式，分别覆盖本地调用、协议接入与平台集成场景。
```

### 4. 接入图片时回答一个明确问题
每张图都应该有职责，例如：
- 这张图解释整体架构
- 这张图解释租户隔离
- 这张图解释调用链路
- 这张图解释能力层与接入层关系

如果图片只是“存在”，没有解释上下文，就仍然算未接入。

### 5. 中文友好化的边界

推荐中文化：
- 标题
- 段落说明
- 操作步骤
- 注意事项
- 设计原则

保留英文：
- 命令
- 路径
- 接口地址
- 配置项
- 类名、函数名、字段名

### 6. 文档命名纠偏
如果文档内容和文件名不一致，优先做以下动作：
1. 判断是内容跑偏，还是命名过期
2. 选择“改正文”还是“改文件名”
3. 同步修复引用链接
4. 检查 README / docs 首页 / 目录索引是否还指向旧文件

## 三、README 质量检查表

满足越多，越接近正式开源项目门面：

- 有清晰标题和一句话介绍
- 能快速说明项目解决的问题
- 有简洁的核心能力概览
- 有快速开始
- 有文档导航
- 有示例入口
- 有贡献、许可证、致谢
- 不依赖读者先翻 docs 才知道项目是什么
- 不把 README 写成实现细节说明书
- 不出现大量时态化的迭代描述

## 四、docs 质量检查表

- 有 `docs/README.md` 或等价入口
- 文档之间有交叉链接
- 一篇文档一个主题
- 文件名和正文语义一致
- 图片有落点、有解释
- 指南类和参考类内容没有混成一锅
- 命令、路径、截图、示例与当前版本一致

## 五、常见反模式

### 反模式 1：把 README 当发布说明
README 应该稳定面向未来，而不是记录这周做了什么。

### 反模式 2：为了重写而删掉原有高质量内容
好的文档改造通常是“保留 + 重组 + 精修”，不是推倒重来。

### 反模式 3：图片目录很完整，但文档里完全没有
图不是资产清单，接入正文后才是有效资产。

### 反模式 4：examples 目录已经更新，README 还在写旧命令
这会直接损害可用性和可信度。

### 反模式 5：全文档英文堆砌
如果项目主要面向中文团队，这种写法会显著拉高理解成本。

## 六、推荐提示语模板

### 模板 1：重构 README
“请把这个项目的根 README 重构成优秀开源项目风格。先盘点现有文档与图片资产，保留高质量内容，再输出正式的项目门面文档，不要写成迭代过程说明。”

### 模板 2：整理 docs
“请为这个项目建立稳定的 docs 信息架构。按入口页 + 专题页整理，并修复文档命名与正文语义漂移问题。”

### 模板 3：全文档纠偏
“请把 README、docs 和 examples 一起同步，完成图片接入、中文友好化、旧命令残留清理和许可证/致谢保留。”

## 七、交付时建议汇报的内容

完成后，建议至少向用户汇报：
- 根 README 是否重构完成
- docs 信息架构是否整理完成
- 图片和静态资产是否已接入
- examples 是否已同步
- 是否做了命名纠偏与旧文案清理
- 当前还剩哪些可选优化项
